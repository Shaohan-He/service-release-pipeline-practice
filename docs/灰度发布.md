# 灰度发布

本项目不使用 Argo Rollouts，而是使用 Kubernetes 原生 Deployment 和 Service 模拟轻量 canary。

## 资源划分

stable：

- Deployment: `release-demo-service`
- Service: `release-demo-service`
- Label: `track: stable`

canary：

- Deployment: `release-demo-service-canary`
- Service: `release-demo-service-canary`
- Label: `track: canary`

stable Service 只选择 `track: stable` 的 Pod，canary Service 只选择 `track: canary` 的 Pod，避免 canary 流量混入 stable。

## 发布步骤

1. 部署 canary Deployment。
2. 等待 canary rollout 成功。
3. 对 canary Service 执行 smoke test。
4. 如果 canary 通过，更新 stable Deployment。
5. 对 stable Service 执行 smoke test。
6. 删除或保留 canary 资源进入观察窗口。

## 验证失败处理

如果 canary 验证失败：

```bash
kubectl -n service-production delete deployment/release-demo-service-canary
kubectl -n service-production delete service/release-demo-service-canary
```

此时 stable 版本没有被更新，用户流量仍然指向上一稳定版本。
