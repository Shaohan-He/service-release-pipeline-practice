# 回滚流程

回滚目标是快速恢复到已知可用版本，并保留清晰证据。

## 模式一：rollout undo

适用场景：

- 最近一次 Deployment revision 可用。
- 需要快速恢复上一版本。
- 不需要指定更早的版本。

命令：

```bash
bash scripts/rollback.sh production --undo
```

## 模式二：指定 tag 回滚

适用场景：

- release record 已记录上一稳定版本。
- 需要回滚到明确版本，例如 `v1.0.0`。
- Deployment revision 历史不完整或不可信。

命令：

```bash
bash scripts/rollback.sh production v1.0.0
```

## 根据 release record 回滚

在 `release-records/` 中找到对应发布记录，查看：

- Previous Version
- Rollback Command
- Commit SHA
- Image

再执行记录中的回滚命令。

## 回滚后检查

回滚完成后必须执行 smoke test：

```bash
bash scripts/smoke-test.sh production
```

还应观察：

- Pod 是否 Ready。
- `/version` 是否显示期望版本。
- `/api/order` 是否恢复正常。
- 日志中是否还有持续异常。
