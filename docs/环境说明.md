# 环境说明

| 环境 | Namespace | Replicas | 镜像 tag 策略 | 自动部署 | 需要审批 | Release Record |
| --- | --- | ---: | --- | --- | --- | --- |
| dev | `service-dev` | 1 | `dev-latest` | 否 | 否 | 可选 |
| staging | `service-staging` | 2 | `staging-latest` 或 commit sha | 是 | 否 | 是 |
| production | `service-production` | 3 | 明确版本号或 digest | 否 | 是 | 是 |

## Dev

dev 用于本地开发和基础验证。默认副本数为 1，镜像 tag 使用 `dev-latest`，适合快速迭代。

## Staging

staging 是预发环境，用于在 production 前验证部署配置、健康检查和服务核心接口。staging 可以自动部署，但仍需要 smoke test 通过后才允许进入 production 提升流程。

## Production

production 是模拟 production namespace，用于演示发布审批、canary、stable 发布和回滚。它不代表真实生产环境。

真实生产环境中，应补充：

- 最小权限 kubeconfig 或 OIDC 工作负载身份。
- Secret 管理和审计。
- 监控告警和日志平台。
- 变更审批记录。
- 发布后观察窗口和应急联系人。
