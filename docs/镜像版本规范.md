# 镜像版本规范

## 版本号 tag

版本号使用语义化形式：

```text
v1.0.0
v1.1.0
v1.2.0
```

这种 tag 用于发布说明、production 部署和回滚记录。

## Commit SHA tag

每次镜像构建同时生成 commit sha tag，例如：

```text
abcdef1
```

它用于把镜像精确追踪到一次代码提交。

## 环境默认 tag

```text
dev-latest
staging-latest
```

这两个 tag 只用于非生产环境的默认配置。它们方便快速演示，但不适合作为 production 发布目标。

## Production 不使用 latest

production 不使用 `latest`，原因是：

- 无法直接看出部署的业务版本。
- 不方便审计和事故复盘。
- 回滚时无法确认上一版本具体镜像。
- 多人协作时容易覆盖同一个 tag。

production 推荐使用明确版本号或 digest。

## 如何确认线上版本

发布后通过 `/version` 查看：

```bash
curl http://localhost:8080/version
```

重点字段：

- `version`
- `git_sha`
- `image_tag`
- `environment`
- `build_time`
